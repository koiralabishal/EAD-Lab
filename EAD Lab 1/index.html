Theory

1. Git Version Control System

  • Distributed version control system for tracking changes
  • Enables multiple developers to work simultaneously
  • Maintains complete history of source code changes

2. Git Repository Structure

  • Working Directory: Where files are edited
  • Staging Area: Preparation for commits
  • Local Repository: Stores commit history
  • Remote Repository: Centralized storage on GitHub

3. Branching and Merging

  • Branches allow parallel development
  • Feature branches isolate new development
  • Merging combines changes from different branches
  • Pull requests facilitate code review

Step 2: Git Repository Setup

  • Initialize the Git repository and configure it with your credentials
  • This step establishes the foundation for version control in your project
# Initialize git repository with proper configuration git init # Configure git with your credentials git config --global user.name "Bishal Koirala" git config --global user.email "your.email@example.com" # Stage the HTML file for initial commit git add index.htm # Create meaningful initial commit git commit -m "feat: Initialize project with index.htm"

Step 3: Remote Repository Integration

  • Establish a secure connection with GitHub using SSH
  • Link your local repository to the remote repository for collaboration
# Generate SSH key with modern encryption ssh-keygen -t ed25519 -C "your.email@example.com" # Add remote repository with SSH URL git remote add origin git@github.com:koiralabishal/EAD-Lab.git # Set main as default branch and push git branch -M main git push -u origin main

Step 4: Branch Strategy Implementation

  • Create a structured branching strategy following GitFlow principles
  • Establish development, feature, and bugfix branches for organized development
# Create and set up development branch git checkout -b develop git push -u origin develop # Create feature branch for new implementations git checkout -b feature git push -u origin feature # Create bugfix branch for potential fixes git checkout -b bugfix git push -u origin bugfix

Step 5: Feature Development and Pull Request Creation

  • Implement new features in the designated branch
  • Create pull requests following proper documentation and review processes
# Switch to feature branch and add enhancements git checkout feature # Add styling improvements to index.htm echo "" >> index.htm git add index.htm git commit -m "feat: Add responsive styling to main page" git push origin feature # Create detailed pull request # Title: "Enhancement: Add Responsive Styling" # Description: "Implemented modern, responsive design elements # - Added custom CSS styling # - Improved typography and spacing # - Enhanced visual hierarchy"

Step 6: Code Review and Merge Process

  • Execute a systematic merge process
  • Ensure code quality and maintain project stability
  • Through proper review procedures
# Merge feature into develop with review git checkout develop git pull origin develop git merge feature --no-ff git push origin develop # Prepare develop for production git checkout main git pull origin main git merge develop --no-ff git push origin main # Clean up completed feature branch git branch -d feature git push origin --delete feature
